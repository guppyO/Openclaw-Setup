# Incident Response

## Immediate actions

1. Freeze spend or automation if treasury or account integrity looks suspicious.
2. Capture logs, screenshots, and affected file paths.
3. Refresh backups if state is still trustworthy.
4. Move risky changes back to stage or lab until the issue is understood.

## Minimum evidence

- timestamp,
- affected agent or subsystem,
- last known good state,
- exact error or external response,
- rollback or recovery action taken.
